## 一、总体判断

整个 notebook 仅以下流水线经历过完整测试且可用：

```
配置Excel → 加载Schema → 加载数据表 → 按主键构建context →
docxtpl渲染(Jinja2变量/条件/循环) → 提取模板批注prompt →
DeepSeek改写 → 输出最终docx
```

其余功能（智能标记系统、RAG知识库、智能分析引擎、Word溯源批注）均**未完成**或**不采用**，不可作为参考。

## 三、核心数据流（可参考的架构）

```
template_relation.xlsx  →  选出模板ID关联的所有表名
entity_schema.xlsx      →  获取主键字段定义
data/*.xlsx             →  加载数据

build_report_contexts(): 合并上述 → (context, TraceMap)
    外层循环: template_ids 列表
    内层循环: 主表每行主键值

render_docx_template() →  中间docx

load_ai_prompts_from_template() + DeepSeekClient
                        →  AI替换§AIBLOCKn§段落

输出最终docx
```
