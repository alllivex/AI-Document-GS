## 一、后端启动（FastAPI + Uvicorn）

### 1. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 启动服务

**方式一（推荐）—— 使用 uvicorn 命令：**

```bash
cd backend
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

- `--reload`：开发模式，代码变化自动重启（可选）
- 监听地址：`http://127.0.0.1:8000`

**方式二——直接运行 Python 模块：**

```bash
cd backend
python -m app.main
```

> 注：方式二不会自动 reload，适合正式运行。

### 3. 验证后端是否启动成功

```bash
curl http://127.0.0.1:8000/api/health
```

或在浏览器打开：http://127.0.0.1:8000/api/health

期望返回 JSON：`{"status": "ok", ...}`

### 4. 查看 API 文档（FastAPI 自动生成）

- Swagger UI：http://127.0.0.1:8000/docs
- ReDoc：http://127.0.0.1:8000/redoc

## 二、前端启动（Vue 3 + Vite）

### 1. 安装依赖

```bash
cd frontend
npm install
```

### 2. 启动开发服务器

```bash
cd frontend
npm run dev
```

- Vite 开发服务器监听地址：`http://127.0.0.1:5173`
- 自动热更新（HMR）

### 3. 验证前端是否启动成功

在浏览器打开：http://127.0.0.1:5173

### 4. 代理说明

前端 Vite 配置了 `/api` 代理：

```
http://127.0.0.1:5173/api/*  →  http://127.0.0.1:8000/api/*
```

因此前端开发时所有 API 请求都通过 `/api` 前缀访问，无需手动配置后端地址。

---

## 三、启动顺序（重要）

1. **先启动后端**（确保 API 可用）
2. **再启动前端**（Vite 启动时会校验一些配置，后端先启动更稳妥）

## 五、常见问题

### Q：端口被占用怎么办？

- 后端：修改 `main.py` 中的 `port` 参数，或命令行指定 `--port 8001`
- 前端：Vite 会自动尝试下一个端口（如 5174），或修改 `scripts/vite-config.mjs` 中的 `server.port`

### Q：后端启动报 `ModuleNotFoundError`

确认已安装所有依赖：

```bash
cd backend
pip install -r requirements.txt --upgrade
```

### Q：前端启动报错

确认 Node.js 版本 ≥ 18，并重新安装依赖：

```bash
cd frontend
rm -rf node_modules
npm install
